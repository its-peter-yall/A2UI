# Phase 13, Plan 01: Resume Course Flow & Last-Active-Node Scrolling

## Objective
Implement the "resume where you left off" experience: when a user clicks Resume
on a CourseCard, the app navigates to the session and auto-scrolls to the exact
node they were last working on. Also add debounced last-active tracking.

@file server/routers/learning.py

## Tasks

### Task 1: Create PATCH /learning/sessions/{id}/last-active endpoint
**Type**: backend
**Files**: `server/routers/learning.py`, `server/database/learning_persistence.py`
**Action**:
1. Add persistence method:
   `update_last_active_node(session_id, node_id)`:
   - UPDATE learning_sessions SET last_active_node_id=?, updated_at=now()
   - Verify session exists → 404 if not
2. Add endpoint:
   `PATCH /learning/sessions/{id}/last-active`
   - Body: { node_id: str }
   - Response: { updated: true }
   - 404 if session not found
3. Add `LastActiveRequest` Pydantic schema

**Verify**: `python -m unittest server.tests.test_learning_router`
**Done**: Endpoint updates last_active_node_id correctly

### Task 2: Update LearningPathContainer with initialNodeId support
**Type**: frontend
**Files**: `client/src/features/learning/LearningPathContainer.tsx`
**Action**:
1. Accept new optional prop: `initialNodeId?: string`
2. On mount, if initialNodeId is set:
   - Find the slide index for that node in the nodes array
   - Set carousel position to that index (initial state)
   - Trigger a brief glow highlight animation on the card (Framer Motion:
     boxShadow pulse with Cyber Yellow for 1.5s)
3. If initialNodeId is null/undefined:
   - Keep existing behavior (start at first active, non-completed node,
     or index 0 for new courses)
4. Fallback: if initialNodeId not found in nodes, go to first non-COMPLETED node

**Verify**: `npm run test -- --run`
**Done**: Carousel scrolls to correct node on mount

### Task 3: Update LearningPage to pass initialNodeId
**Type**: frontend
**Files**: `client/src/features/learning/LearningPage.tsx`
**Action**:
1. When session data loads, extract `last_active_node_id` from session
   (if the field exists in the API response — it was added in Phase 09/10)
2. Pass it as `initialNodeId` to LearningPathContainer
3. If this is a resume (not a fresh generation):
   - Show a brief inline banner or toast: "Resuming where you left off..."
   - Auto-dismiss after 3 seconds
4. Update learningApi type to include last_active_node_id on session response

**Verify**: `npm run test -- --run`, `npm run build`
**Done**: LearningPage passes initialNodeId correctly

### Task 4: Add debounced last-active tracking on frontend
**Type**: frontend
**Files**: `client/src/features/learning/LearningPathContainer.tsx`, `client/src/lib/learningApi.ts`
**Action**:
1. Add `updateLastActiveNode(sessionId, nodeId)` to learningApi:
   - PATCH /learning/sessions/{id}/last-active
   - Fire-and-forget (no await on main thread)
2. In LearningPathContainer, debounce carousel changes:
   - On slide change or progress bar click, capture current node_id
   - Debounce (2 seconds): only fire PATCH after 2s of no movement
   - Use useRef to hold timeout ID, clear on new slide change
   - On unmount: flush pending PATCH immediately (fire if pending)
3. This ensures:
   - Rapid carousel swiping doesn't spam the API
   - When user navigates away, the last position is always saved

**Verify**: `npm run test -- --run`
**Done**: Debounced PATCH fires correctly

### Task 5: Write tests

**Verify**: `python -m unittest`, `npm run test -- --run`
