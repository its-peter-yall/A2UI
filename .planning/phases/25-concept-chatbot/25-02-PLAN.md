# 25-02 Frontend Plan: Concept Chatbot UI

## Objective

Build the client-side concept chatbot experience:

- a bottom-right floating action button on the learning page
- a right-side `ChatPanel` drawer animated with `framer-motion`
- streaming chat response handling with `fetch()`
- `X-Chat-Model` settings support
- hover-only heading chat icons in rendered markdown
- selected heading highlighting and multi-select state
- ephemeral client-side chat history

## Must Haves

- `client/src/lib/chatApi.ts` uses `fetch()` for SSE streaming and does not use Axios.
- Provider settings include `chatModel` and `chatModelTitle`.
- Settings renders a second `ModelPicker` for the chat model.
- `LearningPathContainer` renders a Cyber Yellow bottom-right FAB.
- `ChatPanel` slides in from the right and uses about `400px` desktop width.
- `MarkdownRenderer` overrides `h2`, `h3`, `h4`, `h5`, and `h6`.
- Heading icons are visible on hover, toggle selected heading IDs, and selected headings have a yellow highlight.
- Chat history remains client-side and is capped before sending to the backend.
