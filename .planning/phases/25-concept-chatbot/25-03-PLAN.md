# 25-03 Integration Plan: Concept Chatbot

## Objective

Verify the backend and frontend work as one feature, close edge cases, and document completion:

- API contract alignment between `chatApi.ts` and FastAPI
- SSE parsing under chunk boundaries and error conditions
- settings fallback behavior for `X-Chat-Model`
- heading multi-select behavior
- accessibility and responsive behavior for the right drawer and FAB
- final build/test verification for phase completion

## Must Haves

- Client request JSON keys match backend schema exactly: `message`, `history`, `selected_heading_ids`.
- Client headers match backend aliases exactly: `X-Provider-Api-Key`, `X-Model`, and `X-Chat-Model`.
- General FAB chat works without selected headings.
- Heading chat opens the drawer and sends selected heading IDs.
- Closing the drawer does not persist messages to the backend.
- Empty, failed, aborted, and completed streaming states have deterministic UI behavior.
- Backend and frontend test commands pass before summary is written.
