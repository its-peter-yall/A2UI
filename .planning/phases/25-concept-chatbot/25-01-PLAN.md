# 25-01 Backend Plan: Concept Chatbot

## Objective

Add the backend contract for ephemeral concept chat:

- `POST /learning/sessions/{session_id}/nodes/{node_id}/chat`
- SSE streaming via `fastapi.responses.StreamingResponse`
- free-form OpenAI-compatible text generation via `openai.AsyncOpenAI`
- provider resolution from model slug prefix
- `X-Chat-Model` support with fallback to the existing main model header
- server-side prompt history capped to the last 10 messages

## Must Haves

- `server/routers/learning.py` exposes exactly `POST /learning/sessions/{session_id}/nodes/{node_id}/chat`.
- The endpoint accepts JSON body fields `message`, `history`, and optional `selected_heading_ids`.
- The endpoint reads `X-Provider-Api-Key`, `X-Model`, and `X-Chat-Model` headers; `X-Chat-Model` wins when present.
- The server caps supplied chat history to the last 10 messages before building the LLM request.
- The implementation uses `openai.AsyncOpenAI` directly, not Instructor.
- The response media type is `text/event-stream`.
- Test coverage verifies endpoint success streaming, model fallback, missing provider key behavior, invalid session/node behavior, and history capping.
