# Phase 11, Plan 01: Revision Session CRUD & State Management

## Objective
Build API endpoints for creating, listing, fetching, and deleting revision
sessions. A revision session is a separate tracking layer on top of a completed
course that preserves the original completion data.

## Tasks

### Task 1: Implement revision session persistence methods
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Add `create_revision_session(original_session_id, mode)`:
   - Verify original session exists and status = 'completed' → raise if not
   - Count existing revisions → revision_number = count + 1
   - Create revision_sessions record with UUID
   - Get all concept_nodes for original session
   - Create revision_node_progress record for each node (status='pending')
   - Return dict with revision data + node progress list
2. Add `get_revisions_for_session(session_id, limit=20, offset=0)`:
   - SELECT from revision_sessions WHERE original_session_id
   - ORDER BY started_at DESC
   - Return (list[dict], total_count)
3. Add `get_revision_session(revision_id)`:
   - SELECT revision_sessions with JOIN to revision_node_progress
   - JOIN concept_nodes for title and sequence_index
   - Return dict with revision + nodes
4. Add `delete_revision_session(revision_id)`:
   - DELETE from revision_sessions (CASCADE handles node_progress)
   - Return True/False

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: All CRUD operations work correctly

### Task 2: Create revision API endpoints
**Type**: backend
**Files**: `server/routers/learning.py`
**Action**:
1. `POST /learning/sessions/{session_id}/revisions`:
   - Body: RevisionCreateRequest (mode)
   - Call create_revision_session()
   - Return RevisionSessionResponse (201)
   - 400 if session not completed, 404 if not found
2. `GET /learning/sessions/{session_id}/revisions`:
   - Query: limit, offset
   - Return { revisions: [...], total_count: int }
3. `GET /learning/revisions/{revision_id}`:
   - Return RevisionSessionWithProgress
   - 404 if not found
4. `DELETE /learning/revisions/{revision_id}`:
   - Return { deleted: true }
   - 404 if not found

**Verify**: `python -m unittest server.tests.test_learning_router`
**Done**: All 4 endpoints respond correctly

### Task 3: Write comprehensive tests
**Type**: test
**Files**: `server/tests/test_learning_persistence.py`, `server/tests/test_learning_router.py`
**Action**:
1. Persistence tests:
   - Create revision for completed session succeeds
   - Create revision for in_progress session raises error
   - Create revision for non-existent session raises error
   - revision_number increments correctly (1, 2, 3)
   - List revisions returns correct order
   - Get revision includes node progress with titles
   - Delete revision cascades to node progress
   - Both 'full_review' and 'quiz_only' modes work
2. Router tests:
   - POST returns 201 with valid data
   - POST returns 400 for non-completed session
   - POST returns 404 for non-existent session
   - GET list returns paginated results
   - GET single returns full details
   - DELETE returns success, second DELETE returns 404

**Verify**: `python -m unittest`
**Done**: All tests pass, >80% coverage
