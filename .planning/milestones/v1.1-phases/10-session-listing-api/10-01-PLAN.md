# Phase 10, Plan 01: Session Listing Endpoint with Filtering

## Objective
Create `GET /learning/sessions` endpoint that returns a paginated, filterable,
sortable list of learning sessions with progress data. This powers the course
dashboard on the frontend.

## Context
@file server/routers/learning.py
@file server/database/learning_persistence.py
@file server/schemas/learning.py
@file server/tests/test_learning_router.py
@file .planning/ROADMAP.md (Phase 10, Plan 10-01)

## Tasks

### Task 1: Create LearningSessionSummary schema and response models
**Type**: backend
**Files**: `server/schemas/learning.py`
**Action**:
1. Add `LearningSessionSummary` Pydantic model:
   - id: str
   - query: str
   - course_title: str
   - status: str ('in_progress' | 'completed')
   - progress_percent: int (0-100)
   - total_nodes: int
   - completed_nodes: int
   - last_active_node_title: str | None
   - created_at: str (ISO timestamp)
   - updated_at: str (ISO timestamp)
   - completed_at: str | None
   - revision_count: int (default 0)
2. Add `SessionListResponse` model:
   - sessions: list[LearningSessionSummary]
   - total_count: int
   - has_more: bool

**Verify**: `python -m unittest server.tests.test_learning_schemas`
**Done**: Both schemas validate with correct field types

### Task 2: Create persistence method for session listing
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Add `get_sessions_list(user_id=None, status='all', sort_by='updated_at',
   sort_order='desc', limit=20, offset=0)` method:
   - SELECT from learning_sessions
   - JOIN concept_nodes to count total and COMPLETED nodes
   - LEFT JOIN concept_nodes as cn2 ON last_active_node_id = cn2.id for title
   - LEFT JOIN (subquery) to count revision_sessions per session
   - Apply WHERE filters (user_id, status)
   - Apply ORDER BY (sort_by + sort_order)
   - Apply LIMIT/OFFSET
   - Return (list[dict], total_count)
2. Ensure SQL injection safety (parameterized queries only)
3. Handle edge cases: no sessions, all filtered out

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Method returns correct data with all filter combinations

### Task 3: Create the GET /learning/sessions router endpoint
**Type**: backend
**Files**: `server/routers/learning.py`
**Action**:
1. Add `GET /learning/sessions` endpoint:
   - Query params: user_id, status, sort_by, sort_order, limit, offset
   - Validate: limit max 100, sort_by in allowed values
   - Call persistence `get_sessions_list()`
   - Calculate has_more = total_count > offset + limit
   - Return SessionListResponse
2. Add proper summary and description metadata
3. Add error handling (try/except → log → HTTPException)

**Verify**: `python -m unittest server.tests.test_learning_router`
**Done**: Endpoint returns paginated results correctly

### Task 4: Write comprehensive tests
**Type**: test
**Files**: `server/tests/test_learning_router.py`, `server/tests/test_learning_persistence.py`
**Action**:
1. Persistence tests:
   - Empty list returns ([], 0)
   - List with 3 sessions returns all 3 with correct progress
   - Filter status='in_progress' excludes completed
   - Filter status='completed' excludes in_progress
   - Sort by progress_percent ascending
   - Pagination: limit=2 offset=0 → first 2; offset=2 → next
   - has_more = True when more results exist
   - revision_count accurate (0, 1, 2)
2. Router tests:
   - GET /learning/sessions returns 200 with valid response
   - Filter params pass through correctly
   - Invalid sort_by returns 422
   - limit > 100 capped or rejected

**Verify**: `python -m unittest`
**Done**: All tests pass, >80% coverage
