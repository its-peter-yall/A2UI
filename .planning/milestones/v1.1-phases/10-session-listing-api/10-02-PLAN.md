# Phase 10, Plan 02: Progress Tracking Service & Auto-Status Updates

## Objective
Implement automatic progress calculation and session status management that
fires whenever a node transitions. This includes progress percentage updates,
auto-completion detection, last-active-node tracking, and node timestamps.

## Context
@file server/database/learning_persistence.py
@file server/routers/learning.py
@file server/tests/test_learning_persistence.py
@file .planning/ROADMAP.md (Phase 10, Plan 10-02)

## Tasks

### Task 1: Implement progress calculation and auto-status updates
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Add `_update_session_progress(session_id)` private method:
   - COUNT total concept_nodes for this session
   - COUNT concept_nodes WHERE status = 'COMPLETED'
   - Calculate progress_percent = floor(completed / total * 100)
   - UPDATE learning_sessions SET progress_percent, updated_at
   - If progress_percent == 100:
     - SET status = 'completed', completed_at = now()
   - Return updated progress_percent
2. Integrate into existing `transition_node_status()`:
   - After successful status transition, call `_update_session_progress()`
   - Look up session_id from the node's learning_session_id
3. Add `_update_last_active_node(session_id, node_id)` private method:
   - UPDATE learning_sessions SET last_active_node_id = node_id, updated_at = now()
4. Integrate last-active tracking:
   - Call on LOCKED → VIEWING_EXPLANATION transition
   - Call on quiz submission (the user is actively on this node)

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Progress auto-updates on every node transition

### Task 2: Implement node timestamp tracking
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. In `transition_node_status()`, after successful transition:
   - If target is VIEWING_EXPLANATION and node.started_at is NULL:
     SET started_at = now() (first time only)
   - If target is COMPLETED:
     SET completed_at = now()
2. These are additional UPDATE statements within the same method

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Node timestamps set on correct transitions

### Task 3: Create GET /learning/sessions/{id}/progress endpoint
**Type**: backend
**Files**: `server/routers/learning.py`, `server/schemas/learning.py`
**Action**:
1. Add `SessionProgress` Pydantic schema:
   - progress_percent: int
   - status: str
   - completed_nodes: int
   - total_nodes: int
   - last_active_node_id: str | None
   - last_active_node_title: str | None
2. Add `GET /learning/sessions/{id}/progress` endpoint:
   - Fetch session by id
   - Count nodes by status
   - Look up last_active_node title
   - Return SessionProgress
3. Add `get_session_progress(session_id)` persistence method

**Verify**: `python -m unittest server.tests.test_learning_router`
**Done**: Endpoint returns correct progress data

### Task 4: Write comprehensive tests for progress tracking
**Type**: test
**Files**: `server/tests/test_learning_persistence.py`, `server/tests/test_learning_router.py`
**Action**:
1. Progress calculation tests:
   - 0% when no nodes completed
   - 60% when 3 of 5 completed
   - 100% when all completed → status auto-changes to 'completed'
   - completed_at set only on 100%
   - Progress never exceeds 100%
2. Last-active tracking tests:
   - Updates on LOCKED → VIEWING_EXPLANATION
   - Updates on quiz submission
   - Stores correct node_id
3. Node timestamp tests:
   - started_at set once on first VIEWING_EXPLANATION
   - started_at NOT overwritten on subsequent views
   - completed_at set on COMPLETED transition
4. Router tests:
   - GET /sessions/{id}/progress returns correct data
   - 404 for non-existent session

**Verify**: `python -m unittest`
**Done**: All tests pass, >80% coverage on new code
