# Phase 14, Plan 02: Revision History & Performance Comparison

## Objective
Build the RevisionSummaryModal (shown on revision completion), RevisionHistoryList
(shown on CourseCard), and performance comparison UI. These components close the
revision loop by showing users how they performed vs their original attempt.

@file client/src/features/learning/CourseCard.tsx (from Plan 12-01)
@file client/src/lib/learningApi.ts

@file conductor/product-guidelines.md

## Tasks

### Task 1: Build RevisionSummaryModal component
**Type**: frontend
**Files**: `client/src/features/learning/RevisionSummaryModal.tsx`
**Action**:
1. Create RevisionSummaryModal with props:
   - revisionSummary: RevisionSummary
   - onClose: () => void
   - onReviseAgain: () => void
   - onBackToDashboard: () => void
2. Modal content layout:
   - Title: "Revision Complete!" with celebration icon
   - Mode badge: "Full Review" or "Quiz Only"
   - Stats: Topics Reviewed X/Y, Quiz Score X%
   - Comparison section (if comparison data available):
     - "Original Score: X%"
     - "Improvement: +Y% ↑" (green) or "-Y% ↓" (red) or "No change →" (gray)
   - Action buttons: "Back to Dashboard", "Revise Again"
3. Visual design:
   - Dark modal backdrop with blur
   - Glassmorphism modal card
   - Improvement arrow color-coded (green/red/gray)
   - Framer Motion entrance: scale from 0.9 to 1 with opacity fade
4. Accessible: focus trap, Escape to close, aria-modal="true"

**Verify**: `npm run build`, `npm run test -- --run`
**Done**: Modal renders with correct stats and comparison

### Task 2: Build RevisionHistoryList component
**Type**: frontend
**Files**: `client/src/features/learning/RevisionHistoryList.tsx`
**Action**:
1. Create RevisionHistoryList with props:
   - sessionId: string
   - onViewRevision: (revisionId: string) => void
2. Fetch revision list using:
   ```typescript
   const { data } = useQuery({
     queryKey: ['revisions', sessionId],
     queryFn: () => learningApi.getRevisionsList(sessionId),
   });
   ```
3. Each row renders:
   - "#N" revision number
   - Date (formatted: "Feb 14")
   - Mode badge: "Full Review" or "Quiz Only"
   - Score: "100%" with pass/fail indicator
   - Improvement badge: "+20% ↑" or "-5% ↓"
   - Click → calls onViewRevision(revisionId)
4. Sort by date descending (most recent first)
5. Empty state: "No revisions yet" message
6. Collapsible/expandable: starts collapsed, toggle with chevron

**Verify**: `npm run build`
**Done**: Component renders history list with correct data

### Task 3: Integrate RevisionHistoryList into CourseCard
**Type**: frontend
**Files**: `client/src/features/learning/CourseCard.tsx`
**Action**:
1. For completed courses with revision_count > 0:
   - Show "Revised {n} times" expandable section
   - Clicking expands/collapses RevisionHistoryList
   - Expand/collapse with Framer Motion AnimatePresence
2. Clicking a revision row navigates to that revision's summary
   (or opens RevisionSummaryModal with that revision's data)
3. Lazy loading: RevisionHistoryList only fetches when expanded
   (React Query enabled: false until expanded)

**Verify**: `npm run test -- --run`
**Done**: CourseCard shows expandable revision history

### Task 4: Integrate RevisionSummaryModal into RevisionPage
**Type**: frontend
**Files**: `client/src/features/learning/RevisionPage.tsx`
**Action**:
1. Track revision completion state:
   - When revision status becomes 'completed' (from API poll or mutation response)
   - Fetch revision summary: learningApi.getRevisionSummary(revisionId)
   - Show RevisionSummaryModal with summary data
2. Modal actions:
   - "Back to Dashboard" → navigate('/learn')
   - "Revise Again" → create new revision → navigate to new revision URL
3. Invalidate course list on revision completion
   (so dashboard shows updated revision_count)

**Verify**: `npm run test -- --run`
**Done**: Modal appears on completion with correct data
