# Phase 14, Plan 01: Revision Launcher & Revision Mode Components

## Objective
Build the frontend UI for launching revisions from the dashboard and
experiencing course content in revision mode (full_review or quiz_only).
This includes a new route, RevisionPage, RevisionConceptCard, and hooks.

## Tasks

### Task 1: Add revision TypeScript types and API client methods
**Type**: frontend
**Files**: `client/src/types/learning.ts`, `client/src/lib/learningApi.ts`
**Action**:
1. Add TypeScript types:
   - RevisionSessionResponse: id, original_session_id, revision_number, mode,
     status, progress_percent, total_quiz_score_percent, started_at, completed_at
   - RevisionNodeProgressWithDetails: id, node_id, node_title, sequence_index,
     status ('pending'|'reviewed'|'quiz_passed'|'quiz_failed'), reviewed_at
   - RevisionSessionWithProgress: extends RevisionSessionResponse with
     nodes: RevisionNodeProgressWithDetails[]
   - RevisionSummary: revision_id, mode, progress_percent, total_quiz_score_percent,
     nodes_reviewed, nodes_total, quizzes_passed, quizzes_failed, quizzes_total,
     time_spent_seconds, comparison: {original_quiz_score_percent, improvement_percent}
2. Add API client methods:
   - createRevision(sessionId, mode) → POST /learning/sessions/{id}/revisions
   - getRevisionSession(revisionId) → GET /learning/revisions/{id}
   - markNodeReviewed(revisionId, nodeId) → POST .../mark-reviewed
   - submitRevisionQuiz(revisionId, nodeId, optionId, quizIndex?) → POST .../submit-quiz
   - getRevisionSummary(revisionId) → GET .../summary
   - getRevisionsList(sessionId, limit?, offset?) → GET /sessions/{id}/revisions

**Verify**: `npm run build`
**Done**: Types and API methods compile correctly

### Task 2: Create revision hooks
**Type**: frontend
**Files**: `client/src/features/learning/useRevisionSession.ts`, `client/src/features/learning/useRevisionMutations.ts`
**Action**:
1. useRevisionSession(revisionId):
   - useQuery with queryKey ['revision', revisionId]
   - Returns RevisionSessionWithProgress
   - staleTime: 30_000
2. useRevisionMutations(revisionId):
   - markReviewed mutation: calls markNodeReviewed, invalidates revision cache
   - submitRevisionQuiz mutation: calls submitRevisionQuiz, invalidates cache
   - Both include optimistic updates for revision_node_progress status

**Verify**: `npm run build`
**Done**: Hooks compile and use correct query keys

### Task 3: Build RevisionConceptCard component
**Type**: frontend
**Files**: `client/src/features/learning/RevisionConceptCard.tsx`
**Action**:
1. Create RevisionConceptCard with props:
   - node: ConceptNode (original node with content/quiz)
   - revisionMode: 'full_review' | 'quiz_only'
   - revisionProgress: RevisionNodeProgressWithDetails
   - onMarkReviewed: (nodeId) => void
   - onQuizSubmit: (nodeId, optionId, quizIndex?) => void
2. full_review mode behavior:
   - Show explanation markdown content (always visible, no unlock gate)
   - "Mark as Reviewed" button at bottom → triggers onMarkReviewed
   - Quiz section below (interactive, reuses quiz rendering from ConceptCard)
   - Submit answer → onQuizSubmit callback
3. quiz_only mode behavior:
   - Hide explanation markdown (show only topic title as context header)
   - Show quiz immediately (no "I understand" gate)
   - Submit answer → see feedback → advance
4. Status badges on card:
   - 'pending': ⏳ dimmed indicator
   - 'reviewed': ✅ "Reviewed" green badge
   - 'quiz_passed': ✅ "Passed" green badge
   - 'quiz_failed': ❌ "Try Again" red badge
5. No sequential locking — all cards always accessible

**Verify**: `npm run build`, `npm run test -- --run`
**Done**: Component renders correctly for both modes

### Task 4: Build RevisionPage and add route
**Type**: frontend
**Files**: `client/src/features/learning/RevisionPage.tsx`, `client/src/App.tsx`
**Action**:
1. Add route to App.tsx:
   - `/learn/:sessionId/revise/:revisionId` → RevisionPage
2. RevisionPage component:
   - Extract sessionId and revisionId from URL params
   - Fetch original session (for node content/quizzes)
   - Fetch revision session (for revision progress per node)
   - Header: "📖 Revision #{n} — Full Review" or "📝 Revision #{n} — Quiz Only"
   - "← Dashboard" back button
   - Carousel/list of RevisionConceptCard components
   - Revision-specific progress bar (based on revision progress, not node status)
   - No sequential gating (user can freely navigate between all cards)
   - On revision completion (auto-detected when all nodes done):
     Show RevisionSummaryModal (built in Plan 14-02)

**Verify**: `npm run build`, `npm run test -- --run`
**Done**: Revision route works, page loads with correct mode
