# Phase 12, Plan 01: CourseCard Component & Dashboard API Hook

## Objective
Build the CourseCard React component that displays a single course with its
progress, status, and action buttons. Also create the useCourseList React Query
hook and API client method for fetching course lists.

## Context
@file client/src/features/learning/LearningHome.tsx
@file client/src/lib/learningApi.ts
@file client/src/types/learning.ts
@file conductor/product-guidelines.md (visual identity: Cyber Yellow #FFD400)
@file .planning/ROADMAP.md (Phase 12, Plan 12-01)

## Tasks

### Task 1: Add TypeScript types for session listing
**Type**: frontend
**Files**: `client/src/types/learning.ts`
**Action**:
1. Add `LearningSessionSummary` interface:
   - id, query, course_title, status, progress_percent, total_nodes,
     completed_nodes, last_active_node_title, created_at, updated_at,
     completed_at, revision_count
2. Add `SessionListResponse` interface:
   - sessions: LearningSessionSummary[], total_count: number, has_more: boolean

**Verify**: `npm run build` (TypeScript compilation)
**Done**: Types compile without errors

### Task 2: Add API client method for session listing
**Type**: frontend
**Files**: `client/src/lib/learningApi.ts`
**Action**:
1. Add `getSessionsList(params?)` function:
   - params: { status?, sort_by?, sort_order?, limit?, offset? }
   - GET /learning/sessions with query params
   - Returns SessionListResponse
   - Use the standard `api` client (30s timeout sufficient)
2. Export from learningApi

**Verify**: `npm run build`
**Done**: API client compiles and exports correctly

### Task 3: Create useCourseList React Query hook
**Type**: frontend
**Files**: `client/src/features/learning/useCourseList.ts`
**Action**:
1. Create useCourseList hook:
   - Accept options: { status?, sortBy?, limit? }
   - queryKey: ['courses', options]
   - queryFn: learningApi.getSessionsList(mapped params)
   - staleTime: 30_000 (30 seconds)
   - Return useQuery result (data, isLoading, error, etc.)
2. Export as named export

**Verify**: `npm run build`
**Done**: Hook compiles and uses correct query key

### Task 4: Build CourseCard component
**Type**: frontend
**Files**: `client/src/features/learning/CourseCard.tsx`
**Action**:
1. Create CourseCard component with props:
   - session: LearningSessionSummary
   - onResume: (sessionId: string) => void
   - onRevise: (sessionId: string, mode: 'full_review' | 'quiz_only') => void
2. Visual design per product guidelines:
   - Dark card background with subtle glassmorphism border
   - Course title (truncate long titles with ellipsis)
   - Query text in muted color
   - Progress bar: Cyber Yellow (#FFD400) fill, proportional to progress_percent
   - "{completed_nodes}/{total_nodes} topics completed" text
   - "Last active: {title}" if in_progress and last_active_node_title
3. In-progress state:
   - "Resume Course" primary button (Cyber Yellow)
   - "Started: {date}" meta text
4. Completed state:
   - Green checkmark status badge
   - "Revise Course" button
   - "Practice Quizzes" button
   - If revision_count > 0: "Revised {n} times" badge
5. Hover animation: subtle scale-up (1.02) with Framer Motion
6. Responsive: full-width mobile, card-width desktop

**Verify**: `npm run test -- --run`, `npm run build`
**Done**: Component renders both states correctly

### Task 5: Write tests for CourseCard and useCourseList
**Type**: test
**Files**: `client/src/features/learning/CourseCard.test.tsx`, `client/src/features/learning/useCourseList.test.ts`
**Action**:
1. CourseCard tests:
   - Renders in-progress state with progress bar at correct width
   - Renders completed state with green badge
   - "Resume Course" button calls onResume with sessionId
   - "Revise Course" calls onRevise with 'full_review'
   - "Practice Quizzes" calls onRevise with 'quiz_only'
   - Progress bar shows correct percentage text
   - Revision count badge hidden when 0, shown when > 0
   - Last active node title shown for in-progress
2. useCourseList tests:
   - Fetches data with correct query key
   - Passes filter params to API

**Verify**: `npm run test -- --run`
**Done**: All component tests pass
