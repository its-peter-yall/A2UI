# Phase 12, Plan 02: Dashboard Layout Integration into LearningHome

## Objective
Update LearningHome to display the course dashboard with filtering, sorting,
and pagination. Show courses when they exist; preserve the hero experience
for new users with no courses.

## Context

@file client/src/features/learning/CourseCard.tsx (from Plan 12-01)
@file client/src/features/learning/useCourseList.ts (from Plan 12-01)

@file .planning/ROADMAP.md (Phase 12, Plan 12-02)

## Tasks

### Task 1: Build CourseFilter component
**Type**: frontend
**Files**: `client/src/features/learning/CourseFilter.tsx`
**Action**:
1. Create CourseFilter component with props:
   - status: 'all' | 'in_progress' | 'completed'
   - sortBy: 'updated_at' | 'created_at' | 'progress_percent'
   - onStatusChange: (status) => void
   - onSortChange: (sortBy) => void
2. Visual design:
   - Pill-style filter buttons: "All", "In Progress", "Completed"
   - Active pill: Cyber Yellow background (#FFD400) with dark text
   - Inactive pills: dark background with light text
   - Sort selector: dropdown or pill group for sort options
   - "Recent", "Progress", "Date Created" labels
3. Accessible: role="tablist" for filters, aria-selected on active

**Verify**: `npm run build`
**Done**: Component renders with correct pill styles and interactions

### Task 2: Update LearningHome layout with dashboard section
**Type**: frontend
**Files**: `client/src/features/learning/LearningHome.tsx`
**Action**:
1. Fetch courses using useCourseList hook
2. Conditional rendering:
   - If 0 courses (or loading first time): Full hero + TopicInput + "How It Works"
     + Feature cards (existing experience, no changes)
   - If 1+ courses: Compact hero + TopicInput + "Your Courses" section
     + Feature cards at bottom. "How It Works" collapsed behind disclosure.
3. "Your Courses" section:
   - CourseFilter component at top
   - 2-column grid of CourseCard components (responsive: 1-col mobile)
   - "Load More" button if has_more (pagination)
   - Empty filter state: "No {status} courses found" message
4. State management:
   - useState for filter status (default: 'all')
   - useState for sort (default: 'updated_at')
   - useState for pagination offset (reset on filter/sort change)
   - useCourseList({ status, sortBy, limit: 20 })
5. Navigation callbacks:
   - onResume → navigate(`/learn/${sessionId}`)
   - onRevise → create revision (API call) → navigate to revision URL
6. Framer Motion animations:
   - AnimatePresence on course card list
   - Stagger animation on initial load (50ms gap)
   - Layout animation on filter/sort change

**Verify**: `npm run build`, `npm run test -- --run`
**Done**: Dashboard renders with filtering, sorting, pagination
