# Phase 11, Plan 02: Revision Progress Tracking & Quiz Re-attendance

## Objective
Build endpoints for tracking revision node progress (mark-reviewed, submit-quiz)
and calculating revision-level metrics including a performance comparison against
the original course attempt.

## Tasks

### Task 1: Implement revision node progress tracking
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Add `mark_revision_node_reviewed(revision_id, node_id)`:
   - Verify revision exists and mode = 'full_review'
   - Update revision_node_progress SET status='reviewed', reviewed_at=now()
   - Call `_update_revision_progress(revision_id)`
   - Return updated node progress dict
2. Add `submit_revision_quiz(revision_id, node_id, selected_option_id, quiz_index=0)`:
   - Look up original quiz_data for this node (from concept_nodes)
   - Evaluate answer using existing quiz evaluation logic
   - Create quiz_attempt with revision_session_id set
   - Update revision_node_progress:
     - Correct → status='quiz_passed'
     - Incorrect → status='quiz_failed'
   - Call `_update_revision_progress(revision_id)`
   - Return { is_correct, correct_option_id, explanation, revision_node_status }
3. Add `_update_revision_progress(revision_id)`:
   - Count total nodes and nodes with status != 'pending'
   - Calculate progress_percent
   - Count quiz_passed and quiz_failed for total_quiz_score_percent
   - If all nodes done → set revision status='completed', completed_at=now()
   - Update revision_sessions record

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Node actions update progress correctly

### Task 2: Implement revision summary with performance comparison
**Type**: backend
**Files**: `server/database/learning_persistence.py`, `server/schemas/learning.py`
**Action**:
1. Add `RevisionSummary` Pydantic schema:
   - revision_id, mode, progress_percent, total_quiz_score_percent
   - nodes_reviewed, nodes_total
   - quizzes_passed, quizzes_failed, quizzes_total
   - time_spent_seconds (completed_at - started_at, or None)
   - comparison: { original_quiz_score_percent, improvement_percent } or None
2. Add `get_revision_summary(revision_id)`:
   - Aggregate node progress counts
   - Calculate quiz metrics from quiz_attempts WHERE revision_session_id
   - Calculate original quiz metrics from quiz_attempts WHERE
     revision_session_id IS NULL AND node_id IN (session's nodes)
   - Compute improvement_percent (signed: can be negative)
   - Return RevisionSummary

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Summary returns accurate metrics and comparison

### Task 3: Create revision progress API endpoints
**Type**: backend
**Files**: `server/routers/learning.py`
**Action**:
1. `POST /learning/revisions/{revision_id}/nodes/{node_id}/mark-reviewed`:
   - Call mark_revision_node_reviewed()
   - Return RevisionNodeProgress
   - 400 if mode is 'quiz_only'
   - 404 if revision or node not found
2. `POST /learning/revisions/{revision_id}/nodes/{node_id}/submit-quiz`:
   - Body: { selected_option_id, quiz_index? }
   - Call submit_revision_quiz()
   - Return quiz result + revision_node_status
   - 404 if not found
3. `GET /learning/revisions/{revision_id}/summary`:
   - Call get_revision_summary()
   - Return RevisionSummary
   - 404 if not found

**Verify**: `python -m unittest server.tests.test_learning_router`
**Done**: All 3 endpoints work correctly

### Task 4: Write comprehensive tests
**Type**: test
**Files**: `server/tests/test_learning_persistence.py`, `server/tests/test_learning_router.py`
**Action**:
1. Progress tracking tests:
   - mark_reviewed succeeds in full_review mode
   - mark_reviewed rejected in quiz_only mode
   - quiz submission creates attempt with revision_session_id
   - Correct answer → node status 'quiz_passed'
   - Incorrect answer → node status 'quiz_failed'

2. Summary tests:

   - Zero improvement (no change) handled
   - time_spent_seconds calculated from timestamps
3. Router tests:
   - POST mark-reviewed returns 200 with updated node
   - POST submit-quiz returns quiz result
   - GET summary returns complete metrics
   - 400/404 error cases handled
