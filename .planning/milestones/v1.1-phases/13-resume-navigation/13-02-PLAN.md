# Phase 13, Plan 02: Session Switching & Navigation Guards

## Objective
Enable smooth navigation between the course dashboard and active sessions.
Ensure progress saves on navigation, React Query cache stays fresh, and the
multi-session experience feels seamless.

## Tasks

### Task 1: Update LearningPage header navigation
**Type**: frontend
**Files**: `client/src/features/learning/LearningPage.tsx`
**Action**:
1. Change back button text from "← Back" to "← Dashboard"
2. On back button click:
   - Flush any pending debounced last-active PATCH (from Plan 13-01)
   - Navigate to `/learn` (dashboard)
3. Add "New Topic" button in header (if not already present):
   - On click: Navigate to `/learn` with URL param ?new=true
   - Dashboard auto-focuses TopicInput when ?new=true is present
4. Show course title in header for context

**Verify**: `npm run test -- --run`
**Done**: Navigation buttons work correctly with progress saving

### Task 2: React Query cache management for multi-session
**Type**: frontend
**Files**: `client/src/features/learning/LearningPage.tsx`, `client/src/features/learning/useCourseList.ts`
**Action**:
1. Session detail staleTime:
   - Keep existing staleTime for session data (or set to 60_000)
   - This means returning to a session within 1 minute uses cache
2. Course list invalidation:
   - When LearningPage unmounts (user navigates away from session):
     Use useEffect cleanup to call
     `queryClient.invalidateQueries({ queryKey: ['courses'] })`
   - This ensures dashboard always shows fresh progress
3. New course generation:
   - After generating a new course, invalidate ['courses'] list
   - Add to LearningPathContainer's generation success callback
4. Verify:
   - Two sessions in cache don't conflict (different queryKeys)
   - Returning to dashboard after progress shows updated percentage

**Verify**: `npm run test -- --run`
**Done**: Cache stays fresh across navigation

### Task 3: Browser refresh and URL-based state handling
**Type**: frontend
**Files**: `client/src/features/learning/LearningPage.tsx`
**Action**:
1. On browser refresh at `/learn/{sessionId}`:
   - Session loads from API (React Query refetch)
   - initialNodeId from session's last_active_node_id
   - Resume experience works after refresh
2. On `beforeunload` event:
   - Flush pending last-active PATCH (use navigator.sendBeacon or sync XHR)
   - This handles: browser close, refresh, external navigation
3. Edge case: invalid sessionId
   - If API returns 404, show "Course not found" with "← Dashboard" link
   - Don't crash or show blank screen

**Verify**: `npm run test -- --run`
**Done**: Refresh preserves state, beforeunload saves position
