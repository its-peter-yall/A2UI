## Tasks

### Task 1: Animation polish for dashboard and revision
**Type**: frontend
**Files**: Various components in `client/src/features/learning/`
**Action**:
1. CourseCard entrance animation:
   - Stagger: 50ms delay between cards (index * 50)
   - Scale from 0.95 to 1 + opacity 0 to 1
   - Duration: 0.3s ease-out
2. Progress bar fill animation:
   - CSS transition: width 0.5s ease-out
   - Triggers on value change (not initial render)
3. Revision mode visual distinction:
   - Revision cards have a subtle blue/purple left border (vs yellow for original)
   - Header badge color matches mode (blue for review, green for quiz)
4. RevisionSummaryModal entrance:
   - Backdrop: opacity 0 to 0.5 (0.2s)
   - Modal: scale 0.9 to 1 + opacity fade (0.3s spring)
5. Filter transition:
   - AnimatePresence with layout animation on card grid
   - Cards smoothly rearrange when filter/sort changes
6. "Load More" cards:
   - New cards slide up from below (y: 20 to 0) with stagger

**Verify**: Visual inspection + `npm run build` (no build errors)
**Done**: All animations smooth at 60fps

### Task 2: Edge case handling
**Type**: frontend + backend
**Files**: Various
**Action**:
1. Long course titles: CSS text-overflow: ellipsis, max 2 lines
2. Empty dashboard: Motivational hero ("Start your learning journey")
3. Network error on progress update:
   - React Query retry (3 attempts, exponential backoff)
   - Silent failure for last-active PATCH (don't show error toast)
   - Show error toast for course generation failures
4. Session deleted while user is on it:
   - API returns 404 → show "Course not found" state
   - "← Dashboard" button to recover
5. Revision for deleted session: 404 → redirect to dashboard with toast
6. Progress rounding: Math.floor() always, never show 101%
7. Date formatting: Use Intl.DateTimeFormat for locale-aware dates
8. Zero nodes edge case: Don't divide by zero in progress calculation
9. Very fast quiz completion: Debounce prevents duplicate submissions

**Verify**: `npm run test -- --run`
**Done**: All edge cases handled gracefully

### Task 3: Accessibility compliance
**Type**: frontend
**Files**: Various components
**Action**:
1. CourseCard: role="article", tabIndex=0, keyboard Enter for resume/revise
2. CourseFilter: role="tablist", pills have role="tab", aria-selected
3. Progress bar: role="progressbar", aria-valuenow, aria-valuemin=0,
   aria-valuemax=100, aria-label="Course progress: X%"
4. RevisionSummaryModal: aria-modal="true", focus trap, Escape to close
5. Revision badges: aria-label describing status (e.g., "Quiz passed")
6. Screen reader live regions for:
   - Progress updates: aria-live="polite"
   - Revision completion: aria-live="assertive"
7. Focus management:
   - On revision start: focus first card
   - On modal open: focus first interactive element
   - On modal close: focus back to trigger button

**Verify**: Manual keyboard navigation test + lint
**Done**: Full keyboard navigation works
