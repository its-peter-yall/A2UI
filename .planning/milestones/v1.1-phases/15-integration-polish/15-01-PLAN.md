## Tasks

### Task 1: Server-side integration tests
**Type**: test
**Files**: `server/tests/test_session_lifecycle.py` (new file)
**Action**:
1. Create `TestMultiCourseLifecycle` class (~8 tests):
   - test_generate_course_creates_in_progress_session
   - test_progress_updates_on_node_completion
   - test_session_auto_completes_when_all_nodes_done
   - test_multiple_sessions_coexist_independently
   - test_session_list_reflects_accurate_progress
   - test_last_active_node_tracked_on_transitions
   - test_completed_at_timestamp_set_on_completion
   - test_progress_calculation_accuracy (edge: 0%, 33%, 100%)
2. Create `TestRevisionLifecycle` class (~8 tests):
   - test_create_revision_for_completed_session
   - test_cannot_revise_in_progress_session
   - test_full_review_mark_reviewed_and_quiz
   - test_quiz_only_submit_without_review
   - test_revision_auto_completes
   - test_revision_quiz_attempts_separate_from_original
   - test_revision_summary_performance_comparison
   - test_multiple_revisions_increment_number
3. Create `TestEdgeCases` class (~5 tests):
   - test_delete_session_cascades_to_revisions
   - test_concurrent_sessions_no_interference
   - test_progress_never_exceeds_100
   - test_revision_of_session_with_error_nodes_handled
   - test_empty_session_list_returns_valid_response
4. All tests use unittest.TestCase and in-memory or temp SQLite

**Verify**: `python -m unittest server.tests.test_session_lifecycle -v`
**Done**: All ~21 integration tests pass
