# Phase 09, Plan 02: Revision Sessions Table & Quiz Attempt Extensions

## Objective
Create `revision_sessions` and `revision_node_progress` tables for tracking
course revision attempts. Extend `quiz_attempts` with `revision_session_id` to
separate original vs revision quiz data. Add Pydantic schemas for all new entities.

## Context
@file server/database/learning_persistence.py
@file server/schemas/learning.py
@file server/tests/test_learning_persistence.py
@file .planning/ROADMAP.md (Phase 09, Plan 09-02)

## Tasks

### Task 1: Create revision_sessions and revision_node_progress tables
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Add CREATE TABLE for `revision_sessions`:
   - id TEXT PRIMARY KEY
   - original_session_id TEXT NOT NULL (FK → learning_sessions.id)
   - revision_number INTEGER NOT NULL DEFAULT 1
   - mode TEXT NOT NULL DEFAULT 'full_review' ('full_review' | 'quiz_only')
   - status TEXT NOT NULL DEFAULT 'in_progress'
   - progress_percent INTEGER NOT NULL DEFAULT 0
   - total_quiz_score_percent INTEGER (nullable)
   - started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   - completed_at TIMESTAMP (nullable)
   - Index on original_session_id
2. Add CREATE TABLE for `revision_node_progress`:
   - id TEXT PRIMARY KEY
   - revision_session_id TEXT NOT NULL (FK, CASCADE DELETE)
   - node_id TEXT NOT NULL (FK → concept_nodes.id, CASCADE DELETE)
   - status TEXT NOT NULL DEFAULT 'pending'
   - reviewed_at TIMESTAMP (nullable)
   - Index on revision_session_id
3. Call both from `_ensure_tables()`

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Both tables exist with correct schema and indices

### Task 2: Extend quiz_attempts with revision_session_id
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Add `_ensure_quiz_attempts_revision_column()` method:
   - ALTER TABLE quiz_attempts ADD COLUMN revision_session_id TEXT (nullable)
2. Call from `_ensure_tables()`
3. NULL = original learning attempt, non-NULL = revision attempt

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Column exists, existing data unaffected (all NULL)

### Task 3: Create Pydantic schemas for revision entities
**Type**: backend
**Files**: `server/schemas/learning.py`
**Action**:
1. Add `RevisionMode` as Literal['full_review', 'quiz_only']
2. Add `RevisionCreateRequest(mode: RevisionMode = 'full_review')`
3. Add `RevisionSessionResponse(id, original_session_id, revision_number,
   mode, status, progress_percent, total_quiz_score_percent, started_at,
   completed_at)`
4. Add `RevisionNodeProgress(id, revision_session_id, node_id, status,
   reviewed_at)`
5. Add `RevisionNodeProgressWithDetails(id, node_id, node_title,
   sequence_index, status, reviewed_at)` — extends with node metadata
6. Add `RevisionSessionWithProgress` — extends RevisionSessionResponse
   with `nodes: list[RevisionNodeProgressWithDetails]`

**Verify**: `python -m unittest server.tests.test_learning_schemas`
**Done**: All schemas validate correctly with proper types
