# Phase 09, Plan 01: Session Progress & Status Columns

## Objective
Extend `learning_sessions` and `concept_nodes` tables with progress tracking
columns (status, progress_percent, completed_at, last_active_node_id, started_at)
using ALTER TABLE migrations that follow the existing pattern.

## Context
@file server/database/learning_persistence.py
@file server/schemas/learning.py
@file server/tests/test_learning_persistence.py
@file .planning/ROADMAP.md (Phase 09, Plan 09-01)

## Tasks

### Task 1: Add migration methods for session progress columns
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Create `_ensure_session_progress_columns()` method following the pattern of
   `_ensure_quiz_data_columns()`:
   - `status` TEXT DEFAULT 'in_progress'
   - `progress_percent` INTEGER DEFAULT 0
   - `completed_at` TIMESTAMP (nullable)
   - `last_active_node_id` TEXT (nullable)
2. Call this method from `_ensure_tables()`
3. Handle idempotent migration (columns may already exist)

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: All 4 columns exist in learning_sessions after migration

### Task 2: Add migration methods for node timestamp columns
**Type**: backend
**Files**: `server/database/learning_persistence.py`
**Action**:
1. Create `_ensure_node_timestamp_columns()` method:
   - `started_at` TIMESTAMP (nullable)
   - `completed_at` TIMESTAMP (nullable)
2. Call this method from `_ensure_tables()`
3. Handle idempotent migration

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: Both columns exist in concept_nodes after migration

### Task 3: Write unit tests for schema migrations and progress calculation
**Type**: test
**Files**: `server/tests/test_learning_persistence.py`
**Action**:
1. Test: columns exist after `_ensure_tables()` called
2. Test: default values are correct (status='in_progress', progress_percent=0)
3. Test: NULL handling for optional timestamps
4. Test: progress calculation helper:
   - 0/5 = 0%, 3/5 = 60%, 5/5 = 100%
5. Test: completed_at set only when progress = 100%
6. Test: last_active_node_id updates correctly
7. Test: started_at and completed_at on concept_nodes

**Verify**: `python -m unittest server.tests.test_learning_persistence`
**Done**: All new tests pass, all existing tests still pass
