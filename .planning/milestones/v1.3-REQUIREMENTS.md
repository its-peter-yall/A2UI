# Requirements: A2UI

**Defined:** 2026-02-17
**Core Value:** Users can learn any topic through AI-generated retrieval-based learning paths with gated progression that reinforces understanding through active recall.

## v1.2 Requirements

Requirements for milestone v1.2: Dynamic Quiz Generation. Each maps to roadmap phases.

### Schema & Planning

- [ ] **PLAN-01**: User receives a learning path where each topic has a complexity rating (Basic/Intermediate/Advanced) assigned by the AI planner
- [ ] **PLAN-02**: User receives a learning path where each topic has a quiz count (1-5) determined by the AI planner based on topic complexity
- [ ] **PLAN-03**: User receives quizzes whose difficulty is calibrated to the topic's complexity rating, producing higher-quality questions for complex topics

### Quiz Generation

- [ ] **QUIZ-01**: User receives multiple quizzes per topic when the planner assigns quiz_count > 1, generated as a QuizSet in a single batch
- [ ] **QUIZ-02**: User experiences a difficulty gradient across multi-quiz chains following Bloom's taxonomy (Recall -> Application -> Synthesis)
- [ ] **QUIZ-03**: User's generated QuizSets are stored and retrievable through the existing course pipeline without additional API endpoints
- [ ] **QUIZ-04**: User receives quizzes with validated difficulty ordering, preventing out-of-order or uniform difficulty across a multi-quiz chain

### Backend Progression

- [ ] **PROG-01**: User must pass all quizzes in a set to achieve mastery and unlock the next topic (existing logic verified with real multi-quiz data)
- [ ] **PROG-02**: User must complete quiz N before seeing quiz N+1 within a set (sequential enforcement)
- [ ] **PROG-03**: User can retry only the specific failed quiz in a set without restarting the entire quiz chain
- [ ] **PROG-04**: User can regenerate a topic node and receive a new QuizSet matching the original quiz_count (not just a single quiz)

### Frontend & UX

- [ ] **UXUI-01**: User sees quiz chain progress ("Quiz X of Y") when a topic has multiple quizzes
- [ ] **UXUI-02**: User can advance to the next quiz in a chain after passing the current one (onNextQuiz handler functional)
- [ ] **UXUI-03**: User sees a complexity badge (Basic/Intermediate/Advanced) on each topic card in the learning path
- [ ] **UXUI-04**: User sees a difficulty label (Easy/Medium/Hard) on each quiz within a multi-quiz chain

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Quiz Enhancement

- **QENH-01**: User receives quizzes where later questions reference concepts from earlier quizzes in the chain (inter-quiz context injection)
- **QENH-02**: User's quiz count adjusts dynamically based on performance history across the course (adaptive quiz_count override)
- **QENH-03**: User can view analytics dashboard showing quiz performance trends and mastery progress

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Dynamic question banks (multiple questions per difficulty level) | Scope explosion: 15-25 questions per topic. Same-question retry with option reshuffling is pedagogically valid. |
| Timed quizzes | Adds anxiety, accessibility concerns. No pedagogical benefit for mastery-based learning. |
| Partial credit / multi-select questions | Requires schema, scoring, and UI changes across entire quiz pipeline. Marginal benefit. |
| Learner-chosen difficulty | Defeats AI-driven complexity assessment purpose. System is the curriculum expert. |
| Question type variety (fill-in-blank, matching, free-text) | Each type requires own schema, renderer, evaluator, and prompt. Scope explosion for v1.2. |
| Cross-topic synthesis quizzes | Breaks scatter-gather parallelism. Existing revision system handles cross-topic review. |
| Quiz count range beyond 1-5 | 1-5 provides sufficient range for complexity differentiation. Beyond 5 risks user fatigue and dropout. |
