Add a `generate_quiz_set()` method to the `QuizzerAgent` class. This method generates multiple quizzes in a single LLM call using the `QuizSet` response model.

**Method signature:**
```python
async def generate_quiz_set(
    self,
    topic: TopicNode,
    content: str,
    quiz_count: int,
    context: Optional[dict] = None,
) -> QuizSet:
```

**Implementation details:**

1. **Import QuizSet** at the top alongside existing QuizCard import:
   ```python
   from server.schemas.learning import QuizCard, QuizOption, QuizSet, TopicNode
   ```

2. **Add a batch user message builder** `_build_batch_user_message()`:
   - Accepts `topic`, `content`, and `quiz_count` parameters
   - Produces a user message that instructs the LLM to generate exactly `quiz_count` quizzes
   - Specifies the difficulty gradient explicitly based on quiz_count:
     - quiz_count=1: ["medium"] (single quiz, default difficulty)
     - quiz_count=2: ["easy", "hard"]
     - quiz_count=3: ["easy", "medium", "hard"]
     - quiz_count=4: ["easy", "medium", "medium", "hard"]
     - quiz_count=5: ["easy", "easy", "medium", "hard", "hard"]
   - Include the difficulty sequence in the prompt so the LLM knows exactly what difficulty each quiz should be
   - Include topic title, summary, key_terms, and content (same structure as existing `_build_user_message`)

3. **Implement `generate_quiz_set()`:**
   - Call `self._build_batch_user_message(topic, content, quiz_count)` for the user message
   - Call `self._build_topic_context(topic, context)` for context (reuse existing method)
   - Call `self.generate(response_model=QuizSet, user_message=user_message, context=full_context)` — single LLM call
   - Post-process: call `self._fix_quiz_set_option_ids(quiz_set)` to fix all option IDs across all quizzes
   - Log: `f"QuizzerAgent created quiz set: {len(quiz_set.quizzes)} quizzes for topic: '{topic.title}'"`
   - Return the QuizSet

4. **Add `_fix_quiz_set_option_ids()` method:**
   ```python
   def _fix_quiz_set_option_ids(self, quiz_set: QuizSet) -> QuizSet:
   ```
   - Iterate over all quizzes in `quiz_set.quizzes`
   - For each quiz, call existing `self._fix_option_ids(quiz)` to fix individual quiz option IDs
   - Also collect ALL option_ids across the entire QuizSet and check for duplicates
   - If any duplicates found, regenerate the duplicate UUIDs with `uuid.uuid4()`
   - Return a new QuizSet with fixed quizzes

5. **Guard clause:** If `quiz_count == 1`, delegate to existing `generate_quiz()` and wrap in QuizSet:
   ```python
   if quiz_count <= 1:
       single_quiz = await self.generate_quiz(topic=topic, content=content, context=context)
       return QuizSet(quizzes=[single_quiz], current_index=0)
   ```
   This preserves backward compatibility and avoids changing the batch prompt for simple cases.

6. **Update the file header** to mention `generate_quiz_set` in KEY COMPONENTS.

**Do NOT** modify the existing `generate_quiz()` method — it must remain unchanged for backward compatibility.
**Do NOT** modify `QUIZZER_SYSTEM_PROMPT` — the batch instructions go in the user message, not the system prompt, because the system prompt is shared with single-quiz generation.
