Add advanceToNextQuiz mutation that reuses the existing retry-quiz endpoint to transition SHOWING_FEEDBACK → IN_QUIZ after the server has auto-advanced current_index.

Pattern (based on research findings and existing retry mutation):
```typescript
const advanceToNextQuizMutation = useMutation({
  mutationFn: (nodeId: string) => retryQuiz(nodeId),  // Reuses existing retry endpoint

  onMutate: async (nodeId): Promise<MutationContext> => {
    await queryClient.cancelQueries({ queryKey });
    const rollback = optimisticStatusUpdate(
      queryClient,
      sessionId,
      nodeId,
      'IN_QUIZ'  // Transition back to IN_QUIZ state
    );
    return { rollback };
  },

  onError: (error, _nodeId, context) => {
    context?.rollback();
    onError?.(error as Error, 'advanceToNextQuiz');
  },

  onSettled: () => {
    invalidateSession();  // CRITICAL: Refetch node with new current_index
  },
});

// Export convenience function
const advanceToNextQuiz = (nodeId: string) => {
  advanceToNextQuizMutation.mutate(nodeId);
};
```

Add advanceToNextQuiz to the returned object from useLearningMutations hook.

**WHY reuse retry-quiz:** The server already advanced current_index during submit-quiz when is_correct=true and is_mastered=false (see research Pattern 1). The retry-quiz endpoint transitions state back to IN_QUIZ. On refetch, the node will have quiz_set_hidden.current_index pointing to the next quiz.

**IMPORTANT:** Always invalidate session query in onSettled to ensure cache updates with new current_index.
