  <files>server/tests/test_course_orchestrator.py</files>
  <action>
  Add a new test class `TestMultiQuizMasteryIntegration` to server/tests/test_course_orchestrator.py. These tests verify the persistence layer's mastery and progression logic with real multi-quiz data — the kind the orchestrator now produces after Plan 19-01.

  These tests mock at the persistence layer level (LearningManager methods) to verify behavior without a real database, but they use realistic QuizSet data shapes matching what the orchestrator produces.

  **Import additions:**
  ```python
  from server.schemas.learning import QuizCard, QuizOption, QuizSet
  ```

  **Helper: `_make_quiz_set(quiz_count: int) -> QuizSet`:**
  Create a QuizSet with `quiz_count` QuizCards, each with 4 options where option index 0 is correct. Difficulty follows the gradient: quiz_count=1 → ["medium"], quiz_count=2 → ["easy", "hard"], quiz_count=3 → ["easy", "medium", "hard"]. Each quiz has a unique question string like "Question {i+1} of {quiz_count}".

  **Test 1: `test_mastery_requires_all_quizzes_passed` (PROG-01)**
  Verify that mastery checking logic requires ALL quiz indices 0..N-1 to have at least one correct attempt.
  - Create a QuizSet with quiz_count=3
  - Mock `_check_multi_quiz_mastery(node_id, 3, conn)` call behavior:
    - With correct attempts at indices [0, 1] only → returns False (not all passed)
    - With correct attempts at indices [0, 1, 2] → returns True (all passed)
  - This tests the persistence layer's `_check_multi_quiz_mastery` contract that the orchestrator relies on
  - Since `_check_multi_quiz_mastery` is a private method, test it by calling the public `check_node_mastery()` method with appropriate mocked quiz_data

  **Test 2: `test_sequential_enforcement_via_current_index` (PROG-02)**
  Verify that quiz progression is sequential — the user sees only quiz at `current_index`.
  - Create a QuizSet with quiz_count=3 and current_index=0
  - Verify: when quiz at index 0 is answered correctly, current_index increments to 1
  - Verify: when quiz at index 1 is answered incorrectly, current_index stays at 1
  - This tests the `update_quiz_set_progress()` contract in the persistence layer
  - Mock `update_quiz_set_progress` to verify it's called with expected parameters

  **Test 3: `test_retry_targets_only_failed_quiz` (PROG-03)**
  Verify that retrying a quiz only re-attempts the current quiz, not the entire chain.
  - After failing quiz at index 1, current_index should remain 1
  - On retry and correct answer, current_index increments to 2
  - Previous correct attempt at index 0 is preserved
  - Mock the persistence layer calls to verify retry behavior

  These tests are mock-based verification of the contracts between orchestrator and persistence layer. They prove the data shapes flowing through the system will trigger the correct mastery/progression behavior.

  IMPORTANT: These are NOT end-to-end tests with a real database. They verify the orchestrator produces data that the persistence layer's existing methods handle correctly. The persistence layer methods are already unit-tested in Phase 16 — these tests verify the INTEGRATION point.
