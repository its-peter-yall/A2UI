Update the PlannerAgent's system prompt to instruct the LLM to assess topic complexity and assign appropriate quiz counts for each topic in a learning path.

Purpose: The Planner currently generates TopicNodes without complexity or quiz_count values, so they always default to "Intermediate"/1. </objective>

  Modify the PLANNER_SYSTEM_PROMPT constant in server/agents/planner.py to add complexity assessment and quiz_count mapping instructions. The prompt already defines TopicNode output — the LLM just needs to be told HOW to populate the complexity and quiz_count fields.

  Add these sections to PLANNER_SYSTEM_PROMPT (insert AFTER the "## Decomposition Guidelines" section and BEFORE "## Output Requirements"):

  **Section: "## Complexity Assessment"**
  Explain how to rate each topic's inherent complexity:
  - **Basic**: Vocabulary, definitions, straightforward facts, introductory concepts that establish terminology. Example: "What is a force?" or "Defining key terms."
  - **Intermediate**: Processes, cause-and-effect relationships, comparisons, multi-factor concepts. Example: "How does acceleration relate to force and mass?" or "Comparing elastic vs inelastic collisions."
  - **Advanced**: Deep synthesis, multi-step reasoning, counter-intuitive concepts, abstract theory requiring integration of multiple prior topics. Example: "Deriving orbital mechanics from Newton's laws" or "Quantum tunneling paradoxes."

  Emphasize that a well-designed learning path should have VARIED complexity — typically starting with Basic foundational topics, progressing through Intermediate process topics, and ending with Advanced synthesis topics. Not all topics should have the same rating.

  **Section: "## Quiz Count Mapping"**
  Define the mapping from complexity to quiz_count:
  - **Basic** → `quiz_count: 1` — Single recall quiz sufficient for definitions and facts
  - **Intermediate** → `quiz_count: 2` or `quiz_count: 3` — Multiple quizzes test understanding of processes and relationships
  - **Advanced** → `quiz_count: 3`, `quiz_count: 4`, or `quiz_count: 5` — Progressive quiz chain tests depth: recall → application → synthesis

  State: "The quiz_count determines how many assessment questions the learner must pass before mastering the topic. Higher counts create a difficulty gradient following Bloom's taxonomy (Recall → Application → Synthesis)."

  **Update "## Output Requirements":**
  In the existing "Each TopicNode must include:" list, ADD two new bullet points:
  - `complexity`: Complexity rating (Basic, Intermediate, or Advanced) based on the assessment criteria above
  - `quiz_count`: Number of quizzes (1-5) based on the quiz count mapping above

  **Update "## Example Decomposition":**
  Add complexity and quiz_count to EACH topic in the existing Newtonian Laws example:
  - Index 0 "Forces and Motion Fundamentals" → complexity: "Basic", quiz_count: 1
  - Index 1 "Newton's First Law: Inertia" → complexity: "Basic", quiz_count: 1
  - Index 2 "Newton's Second Law: F=ma" → complexity: "Intermediate", quiz_count: 2
  - Index 3 "Newton's Third Law: Action-Reaction" → complexity: "Intermediate", quiz_count: 2
  - Index 4 "Free Body Diagrams" → complexity: "Advanced", quiz_count: 3
  - Index 5 "Real-World Applications" → complexity: "Advanced", quiz_count: 4

  IMPORTANT: Do NOT change the existing sections (KLI Framework, Decomposition Guidelines, etc.). Only ADD the new sections and UPDATE the example/output requirements. The existing prompt structure works well — we're extending it, not rewriting it.
