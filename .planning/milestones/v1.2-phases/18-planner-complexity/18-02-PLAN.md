Purpose: Even with good prompt engineering (Plan 18-01), LLMs can produce uniform or nonsensical outputs. This will be consumed by the CourseOrchestrator in Phase 19 to decide whether to accept or retry planner output.

Output: validate_complexity_distribution() function in planner.py + comprehensive tests.

  Add a standalone function `validate_complexity_distribution()` to server/agents/planner.py AFTER the PlannerAgent class definition but BEFORE the singleton instance. This is a pure function (not a method) because it validates output, not generates it.

  Function signature:
  ```python
  def validate_complexity_distribution(
      outline: CourseOutline,
  ) -> dict:
  ```

  Import CourseOutline at the top of the file (it's already imported — verify it's in the existing import from server.schemas.learning).

  The function accepts a CourseOutline and returns a dict with:
  ```python
  {
      "valid": bool,           # True if distribution is acceptable
      "warnings": list[str],   # Non-blocking issues
      "errors": list[str],     # Blocking issues
      "distribution": dict,    # {"Basic": N, "Intermediate": N, "Advanced": N}
  }
  ```

  Validation rules:

  **Errors (valid=False):**
  1. **Uniform complexity**: All topics have the same complexity rating. Error message: "All {N} topics have complexity '{X}' — expected varied distribution"
  2. **Quiz count out of range for complexity**: A topic has quiz_count outside its complexity band:
     - Basic with quiz_count > 1: error (Basic should always be 1)
     - Advanced with quiz_count < 3: error (Advanced should be 3-5)
     Note: Intermediate allows 2-3 so is more flexible

  **Warnings (valid=True but flagged):**
  1. **Skewed distribution**: More than 80% of topics share the same complexity. Warning: "{percent}% of topics are '{X}' — consider more variety"
  2. **Intermediate quiz_count at boundary**: Intermediate topic with quiz_count=1 (low for intermediate). Warning: "Topic '{title}' is Intermediate but has quiz_count=1"

  Build the distribution dict by counting topics per complexity level.

  Add a proper docstring following project conventions (summary, Args, Returns, example).

  Do NOT modify PlannerAgent class or PLANNER_SYSTEM_PROMPT (those are from Plan 18-01).
